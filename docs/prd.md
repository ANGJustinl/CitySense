# PRD: Project CitySense（城市脉搏）

CitySense 的目标不是简单做地点搜索，而是构建一个城市实时信号驱动的 AI 推荐系统。

产品通过 MCP / 平台爬虫 / 开放 API 收集城市事件与社交热度，通过高德 API 判断实时可达性，再结合用户兴趣画像生成城市探索路线。

黑客松阶段的核心闭环：

```txt
用户输入兴趣、心情、位置、时间窗口
        ↓
系统从城市事件与地点数据中召回候选
        ↓
结合平台热度与实时交通进行排序
        ↓
生成 3 条城市探索路线
        ↓
用 AI 解释推荐理由、时间安排和出行建议
```

黑客松 demo 必须体现：

1. 用户可以输入偏好并获得 3 条路线。
2. 推荐结果不是普通地点列表，而是可执行路线。
3. 每条路线展示交通耗时、平台来源信号、推荐理由。
4. 高德数据进入推荐排序，而不是只做地图展示。
5. MCP / 爬虫作为城市信号接入层，而不是实时阻塞推荐接口。
