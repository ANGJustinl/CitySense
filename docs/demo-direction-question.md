# 待导师指教：Demo 界面方向选择

> 这是一个需要在动手做 UI 前确定的关键方向。想请导师给个倾向，我再据此推进。

---

## 核心问题

**demo 的首页，应该"让评委一眼看清整个系统的功能和结构"，还是"让用户觉得清新实用、想用"？**

这两个目标会导向完全不同的设计决策，存在取舍。

---

## 两种方向的对比

### 方向 A：直观 / 一眼看全功能（评委导向）

**目标：** 评委 30 秒内理解"这个系统做了什么、技术深度在哪"。

**做法：** 高信息密度，把推荐链路的所有环节都暴露出来——
- 指标条：候选池数量、高德 ETA 调用数、交通重排版本、Top 路线分、缓存命中
- 来源信号：每个地点标注数据来源（高德 POI / 小红书 / 政务公开）
- 用户画像 explain：偏好因子、反感因子、曝光命中
- 多栏并列：偏好输入 + 地图 + 路线详情 + 城市脉冲 + 画像，一屏展示完整链路

**评委感受：** "这背后做了很多，数据链路真实、可追溯。"
**风险：** 评委觉得"信息过载、抓不住重点"，或者"像运维 dashboard 不像产品"。

### 方向 B：清新 / 用户实用（用户导向）

**目标：** 用户觉得"这个我想用"，界面干净、引导清晰。

**做法：** 低信息密度，首屏只露核心交互——
- 偏好输入（兴趣/心情/预算/时间）
- 地图 + 3 条路线卡片
- 技术细节（指标条/来源信号/画像）藏到二级页面或折叠区

**用户感受：** "好懂、想用、不累。"
**风险：** 评委觉得"功能不够多、看不出技术深度"，尤其是"实时信号""可追溯来源"这些卖点被藏住了，评委会怀疑是不是没做。

---

## 两种方向的具体差异

| 维度 | A：直观（评委导向） | B：清新（用户导向） |
|---|---|---|
| 信息密度 | 高——所有数据链路可见 | 低——只露用户关心的 |
| 布局 | 多栏并列（当前 5 列） | 聚焦单一任务流，逐步展开 |
| 视觉风格 | 数据感、工具感（dashboard） | 温暖、留白、引导感（app） |
| 技术细节呈现 | 默认全部展示 | 折叠/点击展开 |
| demo 效果 | "系统很复杂、很真" | "我想用这个" |
| 主要风险 | 信息过载，抓不住重点 | 看不出技术深度 |

---

## 还有一个相关的子问题

当前项目里实际上有**两套首页雏形**，代表两种方向，但它们是割裂的：

| 页面 | 代表的方向 | 当前状态 |
|---|---|---|
| `/`（推荐工作台） | 方向 A（直观/评委） | 真实数据，5 列布局，信息密集，**但偏工具感** |
| `/explore` `/demo` | 方向 B（清新/用户） | mock 数据，兴趣标签云/心情球，**视觉化但和真实系统脱节** |

如果选方向 B，很可能要把 `/explore` 的视觉思路（大标签、情绪化选择、渐进引导）**融合进真实推荐流程**，替代当前的 5 列工作台。

如果选方向 A，则保留工作台，优化视觉层次让重点更突出。

---

## 我的倾向（仅供参考）

**这两个目标其实可以分层实现，不一定要二选一：**

- **首屏默认状态：清新实用**——只露偏好输入 + 地图 + 3 条路线卡片，干净留白
- **展开/细节层：直观**——指标条、来源信号、画像 explain 做成可折叠或点击查看，评委想深挖时一点即开

这样评委看到的是"界面干净但点开有深度"，用户看到的是"好懂想用"。但实现上会比纯选一边稍复杂。

## 想请导师指教的

1. **demo 的主要受众和场景**：是黑客松评委现场评分，还是会公开发布给真实用户用？这决定优先"展示深度"还是"好用"。

2. **方向 A、B、还是分层（A+B）？** 导师觉得哪种更适合当前阶段。

3. **如果选清新方向，/explore 的视觉思路要不要融合进真实首页？** 还是保留两套页面各司其职。

4. **信息密度的底线**：来源信号、交通缓存新鲜度这些"证明推荐真实"的元素，是否必须首屏可见？还是可以藏到点击展开。
